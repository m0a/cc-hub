# CC Hub TUI (embed-tui)

`cchub tui` のローカル専用フルスクリーン端末 UI。**tmux をバックエンド（PTY・端末レンダリング）
としてのみ使い、UI は完全自前描画**する。サーバの HTTP/WS API は経由せず tmux を直接叩く
（`capture-pane` で描画・`send-keys` で入力）ので、CC Hub サーバが起動していなくても動く。

## レイアウト

```
┌─sidebar────┬─────────────────────────────┐
│▸🟡 session │  選択セッションの実端末を       │  左 = 自前サイドバー
│  🔵 other  │  自前領域に描画（capture-pane） │  右 = 選択セッションの実端末
│  🟢 …      │                             │  区切り線の色でフォーカス表示
└────────────┴─────────────────────────────┘
```

## 操作

- **↑↓ / j k / クリック**: セッション選択（サイドバー領域はどこでもクリックでフォーカス）
- **Enter / 右ペインクリック**: 端末フォーカス（打鍵が選択セッションへ転送）
- **Ctrl-B**: サイドバーフォーカスへ戻る
- **n**: 新規セッション（枠線 file manager でフォルダ選択・マウス操作可）
- **H**: 履歴から復帰（`~/.claude/projects` を直読み → `claude -r`）
- **右クリック**（サイドバー行）: コンテキストメニュー → セッションを閉じる
- **`[` `]` / 区切り線ドラッグ**: サイドバー幅調整
- **ホイール**: スクロール（通常=tmux 履歴 / alt-screen=アプリへ転送）
- **q**: 終了

## 実装

- `src/embed/embed-tui.ts` — 本体（自前レンダリング・入力・作成/復帰/メニュー）
- `src/embed/history.ts` — `~/.claude/projects` の履歴読み取り＋resume コマンド生成

エージェント状態ドット（🟡🔴🔵🟢）は、サーバが tmux セッションの `@cchub_state`
オプションに書いた値を読み取る（サーバ未起動時は `·`）。カスタムタイトルは
`~/.cc-hub/session-metadata.json` を直読みする。

## 起動

```bash
cchub tui                 # インストール版バイナリ
bun run dev:tui           # ソースから（= bun run tui/src/embed/embed-tui.ts）
bun --watch tui/src/embed/embed-tui.ts   # 保存で自動再起動
```

実ターミナル（raw mode 対応）が必要。パイプ／ラッパ経由では起動できない。
自分が動いている tmux セッションは一覧から自動除外される（自己参照描画の防止）。
